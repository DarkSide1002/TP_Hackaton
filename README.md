# TP_Hackaton

Проект для автоматизированной обработки корпоративной почты. 
Программа читает письма из папки inbox, определяет категорию каждого письма по ключевым словам, раскладывает файлы по папкам внутри mailbox и формирует отчет со статистикой обработки.

## Возможности

- чтение писем из файловой системы;
- поддержка форматов `.txt`, `.eml`, `.json`, `.html`, `.htm`, `.md`;
- классификация писем по набору правил и ключевых слов;
- обработка пустых, отсутствующих и неподдерживаемых файлов;
- создание папок категорий в `mailbox`;
- сохранение лога обработки в `logs/processing.log`;
- сохранение итоговой статистики в `logs/stats.json`;
- тесты на pytest для основных модулей проекта.

## Категории

Письма распределяются по следующим категориям:

- `spam` - рекламные и подозрительные письма
- `draft` - черновики и письма, которые не нужно отправлять, а также пустые файлы
- `critical` - срочные инциденты и критические проблемы
- `software_issues` - проблемы с программами, доступом и авторизацией
- `hardware_issues` - проблемы с оборудованием
- `financial_requests` - счета, оплаты, договоры и бухгалтерские вопросы
- `meeting_requests` - встречи, созвоны и приглашения
- `informational` - информационные рассылки и уведомления
- `unclassified` - письма без подходящей категории или файлы с ошибками чтения


## Требования

Для запуска приложения нужен Python 3.10 или новее.
Основная программа использует только стандартную библиотеку Python. Для запуска тестов нужен pytest:
Pytest автоматически скачается при запуске bush-скрипта.

## Запуск

Перед запуском убедитесь, что в корне проекта есть папка inbox с письмами.
Запуск через bash-скрипт:

```bash
bash run.sh
```

Скрипт создает папки logs и mailbox, запускает обработку и выводит итоговую статистику из `logs/stats.json`.
Запуск напрямую через Python:

```bash
python3 main.py
```

После запуска результат появится в папке mailbox, а отчет и лог - в папке logs.

## Запуск тестов

```bash
python3 -m pytest tests
```

Тесты используют временные файлы и папки, поэтому не зависят от реального содержимого inbox.

## Отчет

После обработки создается файл:

```text
logs/stats.json
```

В нем сохраняются:
- время запуска;
- продолжительность обработки;
- количество проверенных писем;
- статистика по категориям;
- количество ошибок;
- список файлов, при обработке которых возникли ошибки.
